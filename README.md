# AI в Oil & Gas — сайт-презентация

Скролл-презентация (вместо PowerPoint) о применении AI/ML в нефтегазе и о том, почему «софт в офисе» — самый жизнеспособный путь в российских реалиях. Подготовлено для диалога с [red_mad_robot](https://redmadrobot.ai/).

## О чём рассказ

1. Контекст: у red_mad_robot есть AI/ML, не хватает O&G-домена — это мой вклад.
2. Три области применения AI/ML в нефтегазе:
   - «Хард» на филде (компьютерное зрение, дроны, СИЗ, метан, подсчёты);
   - «Хард» в офисе (ML в геофизике/петрофизике/гидродинамике);
   - «Софт» в офисе (AI-агенты в бизнес-процессах) — **рекомендуемое направление**.
3. 9 обезличенных юзкейсов AI-агентов («ABC company»), из них 3 — выделенные.
4. Три развёрнутых под РФ юзкейса: Закупки, Документы для решений, Cash Calls
   (223-ФЗ/ЕИС, 1С, on-prem LLM, валютный контроль, СПАРК/ФНС и т.д.).
5. Оценки сроков/стоимости MVP в облаке и локальной имплементации (₽, диапазоны).
6. Дорожная карта и предложение сотрудничества.

## Стек

- Vite + React 18 + TypeScript
- Tailwind CSS (тёмная high-tech тема + glassmorphism)
- Framer Motion (scroll-анимации)
- Recharts (графики)
- lucide-react (иконки)

## Локальный запуск

```bash
npm install
npm run dev      # http://127.0.0.1:5173
```

Сборка:

```bash
npm run build    # → ./dist
npm run preview  # http://localhost:4173
```

## Деплой на Render

В репозитории есть `render.yaml` (Blueprint, Static Site):

1. Запушить репозиторий на GitHub.
2. Render Dashboard → **New → Blueprint** → выбрать репозиторий.
3. Render прочитает `render.yaml`: Build `npm ci && npm run build`, Publish `./dist`,
   SPA-rewrite `/* → /index.html`.
4. **Apply** — через 1–2 минуты будет live URL.

## Структура

```
src/
├── App.tsx
├── data/                # весь контент (single source of truth)
│   ├── types.ts
│   ├── areas.ts         # три области применения
│   ├── useCases.ts      # 9 юзкейсов (обезличено: ABC company)
│   ├── deepDives.ts     # 3 развёрнутых юзкейса под РФ
│   ├── estimates.ts     # оценки сроков/стоимости (₽)
│   └── sources.ts       # источники и ссылки
└── components/          # секции скролл-презентации
```

Все цифры и тексты — в `src/data/*`. Оценки сроков/стоимости — экспертные диапазоны
(«оценочно»), не коммерческое предложение.
